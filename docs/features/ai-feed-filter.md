# AI 피드 필터

> RSS 수집 시 AI 관련 기사만 저장하는 키워드 기반 허용 목록 필터.

## 개요

일반 기술 피드(Hacker News, TechCrunch 등)에서 AI와 무관한 기사가 대량 유입되는 문제를 해결한다.
수집 단계에서 제목·설명·태그에 AI 키워드가 포함된 기사만 저장하여, 사이트에 노출되는 피드 품질을 높인다.

## 동작 흐름

```
RSS 피드 수집 → 중복 체크 → 스팸 필터 → AI 키워드 필터 → 저장
```

- 기존 스팸 필터(`isSpam`) 이후 단계에서 `isAIRelated` 허용 목록 필터를 적용한다.
- 제목(title) + 설명(description) + 태그(tags)를 합친 문자열에서 키워드를 대소문자 무시(case-insensitive) 검색한다.
- 키워드가 하나라도 포함되면 AI 관련 기사로 판정하여 저장한다.
- `ai-keywords.json`이 비어 있거나 로드 실패 시 필터를 건너뛴다 (모든 기사 허용).

## 관련 파일

| 파일 | 역할 |
|------|------|
| `src/config/ai-keywords.json` | AI 키워드 허용 목록 (allowlist) |
| `scripts/rss-collect.ts` | `isAIRelated()`, `loadAIKeywords()` 함수 및 수집 로직 |
| `tests/rss-collect.test.ts` | AI 필터 단위 테스트 |

## 설정값

| 이름 | 위치 | 설명 |
|------|------|------|
| AI 키워드 목록 | `src/config/ai-keywords.json` | JSON 문자열 배열. 키워드 추가·제거로 필터 범위 조정 |

## 키워드 관리

- 키워드는 소문자로 비교하므로 대소문자 구분 없이 동작한다.
- 영문·한국어 키워드 모두 지원한다.
- 새 키워드 추가 시 `ai-keywords.json`에 항목만 추가하면 된다. 코드 변경 불필요.

## 제약 사항

- 단순 문자열 포함(substring match) 방식이므로, 짧은 키워드(`ai` 등)가 의도치 않은 단어에 매칭될 수 있다 (예: "email", "maintain"). 현재 제목·설명 맥락에서 이런 오탐이 드물어 허용 가능하다고 판단.
- 키워드 기반이므로 의미론적 판별은 하지 않는다. AI 관련이지만 키워드가 없는 기사는 누락될 수 있다.
